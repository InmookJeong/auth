Auth 애플리케이션 설명::
- Auth는 애플리케이션에서 제공하는 서비스를 이용하기 위해 필요한 인증 기능을 담당합니다.
- Auth는 서비스를 이용할 `회원 정보`와 `이용약관 정보`를 관리하고,
- `권한 관리`를 통해 현재 접속한 사용자가 서비스에 접근하여 이용할 수 있는지 검증합니다.

Auth API 문서 설명::
- Auth API 문서는 Auth 애플리케이션에서 제공하는 서비스를 활용하는 방법을 알려줍니다.
- API 호출을 통해 회원 가입, 로그인, 아이디·비밀번호 찾기 등의 서비스를 이용할 뿐만 아니라
- 로그인 정보를 활용하여 회원 정보 수정, 이용약관 정보 관리, 권한 관리 등의 서비스를 활용할 수 있습니다.
- 프로젝트 버전 0.0.1의 API 문서에서는 한글과 영어를 지원하고 있습니다.

[[이용약관]]
== *2. 이용약관*

설명::
- 이용약관은 애플리케이션의 서비스를 이용하는 과정에서 사용자에게 안내하거나 주의해야 할 정보를 제공하며, 
- 애플리케이션 이용 시 수집되는 정보(개인정보 포함)를 안내하는 서비스입니다.
- 또한 서비스 예정인 애플리케이션의 특성에 따라 이용 약관 정보를 추가·수정·삭제할 수 있습니다.
- 이용약관 정보는 애플리케이션 관리자에 의해 추가·수정·삭제할 수 있으며,
- 사용자는 이용약관 내용을 읽고 회원가입 시 약관에 동의하거나 비동의할 수 있습니다.

==== (1) 잘못된 이용약관 번호(TermsNO)가 전달되었을 경우

설명::
- 이용약관 정보를 상세 조회할 때 잘못된 이용약관 번호(TermsNo), 즉 0이하의 숫자가 전달되면 오류가 발생합니다.
- 이 때 *HTTP Status Code* 중 잘못된 요청(Bad Request)을 의미하는 `*400*` 코드가 반환되며,
- 서비스 이용자가 전달한 언어 정보(Locale)에 따라 오류 메시지를 반환합니다.

==== (2) 올바른 이용약관 번호(TermsNO)가 전달되었지만 이용약관 데이터를 찾을 수 없는 경우

설명::
- 이용약관 정보를 상세 조회할 때 이용약관 번호(TermsNo)는 형식에 맞으나 실제 데이터를 찾지 못할 경우 오류가 발생합니다.

- 서비스 이용자가 전달한 언어 정보(Locale)에 따라 오류 메시지를 반환합니다.
